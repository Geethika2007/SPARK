-> RainShield – Income Protection for Delivery Workers

-> Problem Statement

Delivery partners working with platforms like Swiggy and Zomato depend on daily earnings.
During heavy rain, they are unable to work, leading to significant income loss.

Currently, there is no simple system to protect their earnings during such disruptions.

-> Our Solution

RainShield is an AI-powered parametric insurance system that provides **automatic income protection** to delivery workers.

* Workers pay a small **weekly premium**
* The system monitors external conditions (rain)
* When disruption occurs → **automatic payout is triggered**
* No manual claim required

-> Target Users

* Food delivery partners
* Gig workers dependent on daily earnings

-> Weekly Premium Model

* **Basic Plan:** ₹49/week
* **Pro Plan:** ₹79/week

This aligns with the weekly earning cycle of gig workers.

->Parametric Trigger

* Condition: Heavy rain for more than 3 hours
* System automatically detects disruption

-> Payout Model

* Average earning: ₹100/hour
* Loss assumed: 5 hours
* **Payout: ₹500 per event**
* Maximum payout: ₹500/week

-> How It Works

1. Worker registers on the platform
2. Chooses and activates a weekly plan
3. System monitors weather conditions
4. Rain detected → payout automatically processed

-> Adversarial Defense & Anti-Spoofing Strategy

# 1. Differentiating Real vs Fake Workers

* Genuine users show continuous movement and delivery activity
* Fraud users may show sudden GPS jumps and no delivery activity

# 2. Data Points Used

* GPS location tracking
* Movement patterns
* Delivery/order activity
* Login behavior
* Device ID consistency

# 3. Handling Suspicious Cases

* Claims are flagged for verification
* Payout is temporarily paused
* Genuine users are approved
* Fraudulent users are blocked

This ensures fairness while preventing misuse.
